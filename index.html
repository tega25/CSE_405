Nagios

Nagios is an open source monitoring system for computers as individual nodes or to monitor a network as a whole. It can monitor systems running Linux, Windows, and Unix operating systems. Nagios software runs periodc checks on server and node resources. Memory usage, disk usage, number of processes running, log files etc. are some of the resources that it can monitor. Nagios can also monitor services that running throughout the network or on a individual node. Nagios has command line interface as well as a graphical user interface. Nagios can send out alerts in real time if resource usage reaches a critical state determined by the administrator or if services begin failing. These alerts can be send via email or text or to another server.

Research

I installed nagios on a VM. I found it was bit tricky because it was not just one installation file. I had to first install the necessary dependencies. After installing Nagios I tried to start to the nagios service. I repeatedly ran into an error. I researched the issue and learned that I had to create a new user and group to run the Nagios processes. Allowing Nagios processes to be ran as the current user is an option. After that I had to do the rest of the configuration in the nagios conf files. Then I had to install the Nagios Remote Plugin Executor on the host that I wanted to monitor. I learned how that with nagios I can monitor vital resources that are limited in a real workplace setting. Nagios could help with maximizing resources, extending lifespan of resources and ensuring that your services are continousally available.
